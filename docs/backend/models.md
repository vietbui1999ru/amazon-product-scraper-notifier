# Data Models

Three tables managed by SQLAlchemy ORM, schema changes applied via Alembic.

---

## Product

Table: `products`

| Column | Type | Nullable | Default | Notes |
|---|---|---|---|---|
| `id` | `Integer` | no | autoincrement PK | |
| `url` | `String(2048)` | no | — | UNIQUE constraint |
| `name` | `String(500)` | no | — | |
| `asin` | `String(20)` | yes | NULL | Extracted from `/dp/{ASIN}` in URL |
| `image_url` | `String(2048)` | yes | NULL | Added in migration 001 |
| `rating` | `String(50)` | yes | NULL | Raw string from Amazon (e.g. `"4.5 out of 5 stars"`) — added in 001 |
| `created_at` | `DateTime(timezone=True)` | no | `func.now()` | Server-side default |

**Relationships**

- `price_checks` — one-to-many to `PriceCheck`. Cascade: deleting a `Product` deletes all associated `PriceCheck` rows (`ondelete="CASCADE"`).

**Constraints**

- `UNIQUE(url)` — enforced at DB level. `get_or_create_product` relies on this to stay idempotent.

## PriceCheck

Table: `price_checks`

| Column | Type | Nullable | Default | Notes |
|---|---|---|---|---|
| `id` | `Integer` | no | autoincrement PK | |
| `product_id` | `Integer` (FK → `products.id`) | no | — | `ondelete="CASCADE"` |
| `price` | `Numeric(10, 2)` | yes | NULL | NULL when scrape failed |
| `currency` | `String(3)` | no | `"USD"` | Three-letter code |
| `scraped_at` | `DateTime(timezone=True)` | no | `func.now()` | Server-side default |
| `scrape_success` | `Boolean` | no | `False` | `True` only when price was extracted |
| `error_message` | `Text` | yes | NULL | Populated when `scrape_success=False` |
| `notified` | `Boolean` | no | `False` | Set to `True` after drop notification sent |
| `notification_lock` | `Boolean` | no | `False` | Reserved column — not used by current notification logic (lock is via `pg_try_advisory_xact_lock`) |
| `source` | `String(20)` | no | `"amazon"` | Added in migration 002; see values below |

**`source` values**

| Value | Meaning |
|---|---|
| `amazon` | Price scraped live from Amazon via Playwright |
| `self` | Price injected manually — via `POST /api/demo/drop`, `demo_drop.py`, or `POST /api/products` with `initial_price` |
| `simulated` | Price generated by `simulate_price_walk.py` for chart demo purposes |

**`scrape_success=False` rows**

When the scraper fails (blocked, selector not found, timeout), a `PriceCheck` row is still written with `scrape_success=False`, `price=NULL`, and `error_message` set. These rows appear in history queries but are excluded from `get_last_successful_price` and `get_previous_successful_price`.

**`notified` and `notification_lock`**

`notified` is set to `True` after the notifier fires. The scheduler uses `pg_try_advisory_xact_lock` at the Postgres level (not `notification_lock`) to prevent duplicate notifications in concurrent scenarios. `notification_lock` is a reserved column that is currently unused.

## ScheduledPrice

Table: `scheduled_prices`

Created in migration 003.

| Column | Type | Nullable | Default | Notes |
|---|---|---|---|---|
| `id` | `Integer` | no | autoincrement PK | |
| `product_id` | `Integer` (FK → `products.id`) | no | — | `ondelete="CASCADE"` |
| `price` | `Numeric(10, 2)` | no | — | Target price to inject |
| `currency` | `String(3)` | no | `"USD"` | Server-side default |
| `scheduled_for` | `DateTime(timezone=True)` | no | — | UTC timestamp when price should be applied |
| `created_at` | `DateTime(timezone=True)` | no | `func.now()` | Server-side default |
| `applied_at` | `DateTime(timezone=True)` | yes | NULL | Set when scheduler applies the row |
| `cancelled_at` | `DateTime(timezone=True)` | yes | NULL | Set when row is cancelled |
| `cancel_reason` | `String(100)` | yes | NULL | See reason values below |

**Pending invariant**

A row is "pending" when both `applied_at IS NULL` AND `cancelled_at IS NULL`. The `_pending_filter()` helper in `ProductRepository` encodes this. Rows that have either column set are settled and will not be re-applied.

**`cancel_reason` values**

| Value | Source |
|---|---|
| `"amazon_scrape"` | A real Amazon scrape completed before `scheduled_for` — cancels all pending rows for that product |
| `"manual"` | Cancelled via `DELETE /api/scheduler/prices/{id}` |

## Alembic Migration Chain

```
001_add_product_metadata  →  002  →  003
```

| Revision ID | Revises | What it adds |
|---|---|---|
| `001_add_product_metadata` | (none — initial) | `products.image_url VARCHAR(2048)`, `products.rating VARCHAR(50)` |
| `002` | `001_add_product_metadata` | `price_checks.source VARCHAR(20) NOT NULL DEFAULT 'amazon'` |
| `003` | `002` | Creates `scheduled_prices` table with all columns listed above |

All migrations use `IF NOT EXISTS` / Alembic `op` calls and include `downgrade()` functions.
